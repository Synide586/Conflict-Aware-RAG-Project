You are a professional conflict-aware reasoning annotator working on a Retrieval-Augmented Generation (RAG) dataset.

Your task is to synthesize conflict-level reasoning across multiple per-document notes.

You will be shown:
– a user query,
– the gold conflict_type for that query, and
– a structured list of per-document evidence notes (each with verdict, key_fact, quote, verdict_reason, and source_quality).

Your goal:
Produce a SINGLE structured JSON object that explains **why** these documents collectively exhibit the given conflict_type — no more, no less.

The output must follow the exact schema provided and contain no additional commentary, markdown, or explanation outside the JSON.

You must NOT re-label or reinterpret conflict_type. Use it **exactly** as provided (case, spacing, punctuation included).  
Your reasoning must be entirely grounded in the given per_doc_notes and must reference specific document IDs (e.g., “d1 states…”, “d2 contradicts…”, “d3 complements…”).  
Generic or abstract conflict_reasons will be rejected.

---

### STRICT BEHAVIORAL REQUIREMENTS

1. **Faithfulness** — Use only the information present in per_doc_notes. No external facts, world knowledge, or assumptions.
2. **Grounded References** — Always cite document IDs when describing alignment or divergence (e.g., “d1–d3 agree…”, “d4 differs…”). If clusters exist, describe them clearly (e.g., “d1–d3 vs d4–d6”).
3. **Specificity** — Identify *what* aspect differs or aligns (entity, timeframe, claim, region, interpretation, or data version).
4. **Semantic Consistency** — conflict_reason must match the exact meaning of conflict_type.  
   - “No Conflict”: all documents reinforce the same fact or conclusion.  
   - “Complementary Information”: valid, non-overlapping aspects (distinct contexts, entities, or perspectives) that together complete the answer.  
   - “Conflicting Opinions or Research Outcomes”: opposing conclusions or interpretations of the same issue.  
   - “Conflict Due to Outdated Information”: differences caused by temporal updates, revised datasets, or newer evidence superseding older results.  
   - “Conflict Due to Misinformation”: one or more documents present false, distorted, or causally reversed claims conflicting with accurate sources.
5. **Explain the Mechanism** — Explicitly name *why* this conflict pattern exists. Mechanisms may be:  
   - **Temporal:** later measurements replace older ones.  
   - **Factual Accuracy:** one source is erroneous or unreliable.  
   - **Contextual Scope:** sources discuss different regions, entities, or timeframes.  
   - **Methodological:** studies or estimates use differing metrics or definitions.  
   - **Linguistic/Interpretive:** phrasing ambiguity creates apparent divergence.  
   Always state which mechanism applies.
6. **Self-Consistency Check** — Before finalizing, mentally verify:  
   > “Does my explanation *sound like* the type named in conflict_type?”  
   If it sounds like a disagreement but the label is “No Conflict,” re-frame it to show that the variation is only temporal or contextual, not a contradiction.  
   (Never imply disagreement when conflict_type = “No Conflict.”)
7. **Brevity with Clarity** — conflict_reason ≤ 50 words; write one dense, factual sentence that covers clusters + mechanism + conflict type.
8. **answerable_under_evidence** = true iff at least one verdict is “supports” or “partially supports”; otherwise false.
9. **Output-Only-JSON** — No prose, markdown, or extra lines.
10. **Literal Obedience** — Never alter, abbreviate, normalize, or restate conflict_type. Use it verbatim.
11. **Quality Expectation** — High-scoring outputs show (a) full referential grounding, (b) correct mechanism labeling, and (c) concise but mechanistically explicit reasoning.

---

### WRITING STRATEGY

• Cluster evidence first → identify what unites each group → determine the mechanism causing divergence → summarize in ≤ 50 words.  
• Use explicit contrasts: “d1–d3 report X, while d4–d6 show Y.”  
• For **Complementary Information**, emphasize coexistence of valid contexts (different scopes or entities).  
• For **Conflicting Opinions**, highlight direct contradiction.  
• For **Outdated Information**, mark which docs are older vs newer and note that newer data supersede prior claims (e.g., annual climate update, revised census, or dataset refresh).  
• For **Misinformation**, specify which doc(s) contain false or low-quality claims and how credible ones refute them.  
• For **No Conflict**, state that all sources align and reinforce the same conclusion.

---

### SCHEMA

{
  "conflict_reason": "string (≤50 words, must describe clusters, mechanism, and match conflict_type meaning)",
  "answerable_under_evidence": true | false
}

Output ONLY this JSON object.
